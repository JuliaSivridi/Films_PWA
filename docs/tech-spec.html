1. Overview

Films PWA is a personal movie-collection manager. The user authenticates with Google OAuth 2.0; their film data lives in a private Google Spreadsheet with no custom backend required.

Key Design Decisions

  • No backend server. All persistence goes directly to Google Sheets API v4.
  • No routing library. Navigation is React state: view: 'list' | 'stats'.
  • No chart library. The statistics donut is rendered with raw SVG path math.
  • No CSS-in-JS. Styles live in per-component CSS Modules plus global custom properties.
  • Virtual scrolling for the movie list (3 000+ films) via @tanstack/react-virtual.
  • CI-generated PNG icons — rsvg-convert converts the SVG source to 192 and 512 px PNG in the GitHub Actions workflow before Vite build runs.

4. Data Model

4.1 Movie Interface

export type MovieStatus = 'want' | 'watched'

export interface Movie {
  id:            string           // UUID
  title_ru:      string           // Russian title
  title_orig:    string           // Original title
  year:          number           // Release year (0 if unknown)
  status:        MovieStatus
  tmdb_id?:      string
  poster_path?:  string           // TMDB path, e.g. "/abc123.jpg"
  genres?:       string[]         // ["Drama", "Thriller"]
  tmdb_rating?:  number           // rounded to 1 decimal
  duration_min?: number
  kinopoisk_url?: string
  imdb_url?:     string
  tmdb_url?:     string
  wiki_url?:     string
  countries?:    string[]         // ["France", "Italy"]
  keywords?:     string[]         // ["fairy tale", "based on novel"]
  _row?:         number           // 1-based Sheets row (runtime only)
}

export const STATUS_COLORS: Record<MovieStatus, string> = {
  want: '#f59e0b', watched: '#10b981'
}

6. Services Layer

6.1 auth.ts — Token Lifecycle

initAuth(clientId)     → if profile saved → trySilentSignIn()
trySilentSignIn()      → prompt:'', login_hint; sets tokenExpiresAt
signIn()               → prompt:'consent'
refreshTokenIfNeeded() → if Date.now() > tokenExpiresAt − 30_000 → silent refresh
                         called at the top of every Sheets/Drive request

6.2 sheets.ts — parseArr + rowToMovie

function parseArr(raw?: string): string[] | undefined {
  if (!raw) return undefined
  try {
    const parsed = JSON.parse(raw)
    return Array.isArray(parsed) ? parsed : undefined
  } catch { return undefined }
}

// rowToMovie maps 16-column string array → Movie object
// _row = rowIndex + 2  (0-index offset + header row)

6.3 drive.ts

findOrCreateFilmsFile(): checks localStorage cache → searches Drive → creates spreadsheet if missing. listUserSheets(): Drive files list for the Settings picker.

6.4 tmdb.ts

  • searchMovies(q) — up to 8 results, language ru-RU
  • getMovieDetails(id) — append_to_response=external_ids,keywords
  • getGenres() — fetches once, module-level cache
  • getPosterUrl(path, size) — TMDB image CDN
  • formatDuration(min) — "1h 45m"

Status chips · Year range · Rating range · Country substring. "Clear all filters" when activeFilterCount > 0.

Sorted (localeCompare('ru')), grouped into letter dividers + rows. Cyrillic → A–Z → #.

Row content: poster (90×135 px) · titles · year/duration/★rating · countries · KP/IMDb/TMDB/Wiki links.

Status dot: 16×16 px circle, position: absolute bottom-right of the poster button. Colour from STATUS_COLORS[m.status] — green (#10b981) for Watched, amber (#f59e0b) for Want. border: 2px solid var(--surface) separates it visually from the poster.

8.4a MovieGrid

Orchestrator for the list view: loading spinner → error text → empty state → <MovieList> + FAB. Manages editing: Movie | null to open AddMovieModal in edit mode.

Result count — shown above the list when filtered.length > 0. No active filters: "312 films". Filters/search active: "47 of 312" (when filtered.length !== movies.length).

8.5 AddMovieModal

Two-phase ('search' → 'form'). Background enrichment: genres, runtime, countries, keywords, IMDb ID, KP/Wiki fetched concurrently after TMDB pick; currentTmdbId ref discards stale results. Form order: Poster+Titles → Year/Duration → Status → Links → Countries → Genres → Keywords. Default status: 'want'.

8.6 StatsPage

Dimensions: Decade · Rating · Country · Genre. computeSlices() returns { slices, uniqueCount }.

DimensionSourceGrouping
DecadeyearMath.floor(year/10)*10 + 's'
Ratingtmdb_ratingMath.floor(rating) + '.x'

uniqueCount vs. slice sum: The donut centre shows uniqueCount — films that had at least one valid value for the dimension — not the sum of all slice counts, which is inflated for multi-value fields (a film with 3 genres contributes 3 to the sum but only 1 to uniqueCount).

SVG donut: viewBox="0 0 280 280", cx=140, cy=140, R=125, r=68, start angle −π/2. Top 15 slices + Others bucket (#94a3b8).

8.7 SettingsModal

TMDB key (password input → localStorage); spreadsheet picker (listUserSheets() → <select>).

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: 24 }
      - run: npm install
      - name: Generate PNG icons from SVG

          sudo apt-get install -y librsvg2-bin

          rsvg-convert -w 512 -h 512 public/icons/icon.svg > public/icons/icon-512.png
      - run: npm run build
        env: { VITE_GOOGLE_CLIENT_ID: ${{ secrets.VITE_GOOGLE_CLIENT_ID }} }

        with: { path: dist }
  deploy:
    needs: build
    uses: actions/deploy-pages@v4
PNG icons are generated before the Vite build so they land in dist/icons/. librsvg2-bin is installed via apt-get; rsvg-convert correctly renders SVG attributes including fill, stroke, and rx.

12.2 Alphabetical Grouping

function firstLetter(m: Movie): string {
  const ch = (m.title_ru || m.title_orig || '')[0]?.toUpperCase() ?? ''
  if (/[А-ЯЁ]/.test(ch)) return ch   // Cyrillic first
  if (/[A-Z]/.test(ch))   return ch   // then Latin
  return '#'                           // then everything else
}
// Letter order: CYRILLIC.indexOf(l) → 100 + charCode → 999

12.3 SVG Donut Sector Path

function sectorPath(cx, cy, R, r, a1, a2): string {
  const large = a2 - a1 > Math.PI ? 1 : 0
  return [
    `M ${cx + R*cos(a1)} ${cy + R*sin(a1)}`,
    `A ${R} ${R} 0 ${large} 1 ${cx + R*cos(a2)} ${cy + R*sin(a2)}`,
    `L ${cx + r*cos(a2)} ${cy + r*sin(a2)}`,
    `A ${r} ${r} 0 ${large} 0 ${cx + r*cos(a1)} ${cy + r*sin(a1)}`,
    'Z',
  ].join(' ')
  // Outer arc CW (sweep=1), inner arc CCW (sweep=0)
}

12.4 Stats — uniqueCount vs. Slice Sum

// Track films-with-data separately from the per-category counts:
const validKeys = keys.filter(k => !!k)
if (validKeys.length > 0) uniqueCount++    // film counted once
for (const k of validKeys) counts[k]++    // each key counted separately

// Donut centre = uniqueCount (not sum of slice.count)
// A film with genres ["Drama","Comedy","Thriller"] adds 3 to sum but 1 to uniqueCount

12.5 Background Enrichment Race Guard

const currentTmdbId = useRef<string | null>(null)

async function selectTMDB(t: TMDBMovie) {
  const tmdbId = String(t.id)
  currentTmdbId.current = tmdbId
  setPhase('form')              // show form immediately
  // ...fetch genres, details, wikidata...
  if (currentTmdbId.current !== tmdbId) return  // user changed selection
  setForm(f => ({ ...f, ...enrichedData }))
}
